Pre-rendering (SSG)

What is pre-rendering?

Pre-rendering means to render the HTML of all our pages at build-time. Normally, the HTML of a page is rendered at request-time (when the user navigates to that page). But, with pre-rendering, the HTML of a page is rendered at build-time instead (when we call $ vite build).

When using pre-rendering, we don't need a Node.js server: our app consists only of static assets (HTML, JS, CSS, images, ...) and we can deploy these static assets to a so-called "static host" such as GitHub Pages, Cloudflare Pages, or Netlify. If we don't use pre-rendering, then we need a Node.js server to be able to render our pages' HTML dynamically at request-time.

How to pre-render

To pre-render our pages, we use the CLI command prerender at the end of the build:

  • npm: $ npx vite build && npx vite build --ssr && npx vite-plugin-ssr prerender.
  • Yarn: $ yarn vite build && yarn vite build --ssr && yarn vite-plugin-ssr prerender.

The generated static assets are available at dist/client/.

The only difference between using $ vite-plugin-ssr prerender and using createPageRenderer() is when the HTML is rendered (i.e. render()/onBeforeRender() hooks called at build-time VS request-time). Note that client-side code is always executed at request-time (which makes sense because it's executed in the user's browser).

For pages with a parameterized route (e.g. /movie/:movieId), we have to use the prerender() hook in order to provide a list of URLs that are to be pre-rendered.

We can also use the prerender() hook to accelerate the pre-rendering process as it allows us to prefetch data for multiple pages at once.

We can skip pre-rendering for certain pages by using .page.server.js#doNotPrerender.

Should I pre-render?

We should use pre-rendering whenever we can. Because pre-rendering removes the need for a Node.js server which makes deployement an order of magnitude easier (and it's also more performant as HTML isn't re-generated on every request).

The problem is that pre-rendering cannot be used for every kind of website. So the question here is: for what kind of websites can we use pre-rendering?

Pre-rendering cannot be used for websites that have highly dynamic content. For example, a social site such as Hacker News or Reddit has highly dynamic content: the content changes every time a user posts a new link or writes a new comment. We cannot use pre-rendering because the HTML is stuck at build-time and we cannot re-build the entire website every time there is new content.

Pre-rendering works for websites with content that changes only few times a day. For example, https://vite-plugin-ssr.com is updated only occasionally, and we can re-build the entire https://vite-plugin-ssr.com website every time we change its content. Pre-rendering allows us to deploy https://vite-plugin-ssr.com to GitHub Pages which is an order of magnitude easier (and more performant) than using a Node.js production server.
